Where the work happens

Then we shipped Drive. Read-only became the read mode of a thing whose center is now write — starting, steering, and managing agents live.

The control plane

Everything you need to run an agent — none of the code you don't.

Agentic-first coding moved the leverage up a level. You mostly don't read source anymore; you direct. VibeRate gives you the four surfaces that actually matter.

▶

Drive

A composer that spawns a real claude on a bound checkout and steers it turn by turn. Kick off and babysit agents from anywhere — the work runs on the host, not your device.

◈

The brain

Your .md docs, plans, and memory are the real steering wheel — not the source. The brain web makes them legible and glows live as the agent edits them — so you see what's done, what's in flight, and which plans stalled before they quietly fall through the cracks.

◐

Context in view

An agent gets dumber as its window fills. Your harness auto-compacts — but compaction is blunt and happens off-screen. VibeRate keeps fullness in view as you drive, so you can branch or start fresh on purpose instead of letting the window decide for you.

⊞

A fleet, not a session

The unit of work is several agents in flight at once. Start, park, resume, and re-enter the right one — across devices, across redeploys, across repos.

The single-developer loop

Capture → understand → drive. On a phone, frictionless.

One vertical scroll: the live transcript and composer, a brain header strip, context and progress. The cooled history is the same object — prompt units with outcome chips and evidence, ready to review.

01 · CAPTURE

It records itself

Every prompt, diff, commit, command, and screenshot is captured as the agent works — private by default, no extra step.

→
02 · UNDERSTAND

See what landed

Prompt units carry auto-derived outcome chips: files changed, commits, brain docs touched, context fullness. Read the work, not the diff.

→
03 · DRIVE

Send the next move

The transcript's live head is the composer. Steer, branch, or hand off the next prompt — the loop closes without leaving the conversation.
